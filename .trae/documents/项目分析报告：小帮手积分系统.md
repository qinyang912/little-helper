这是一个名为 **"小帮手积分系统" (Little Helper)** 的全栈 Web 应用，旨在通过游戏化的方式帮助家庭管理家务和奖励。

### 1. 项目概览
- **核心目标**：建立一个家庭内部的积分激励系统，孩子通过做家务赚取积分，并用积分兑换奖励。
- **用户角色**：
  - **家长 (Parent)**：管理员角色，负责创建孩子账户、设置家务/奖励、审批任务。
  - **孩子 (Child)**：用户角色，完成家务（需审批）、赚取积分、兑换奖励。

### 2. 技术栈架构
项目采用现代化的前后端分离架构：

#### 前端 (Client)
- **框架**：React 19 + Vite (构建工具)
- **样式**：Tailwind CSS (实用优先的 CSS 框架)
- **UI/交互**：
  - `framer-motion`:用于流畅的动画效果
  - `lucide-react`: 提供图标支持
  - `@use-gesture/react`: 手势交互支持
- **通信**：
  - `axios`: 处理 HTTP 请求
  - `socket.io-client`: 实现实时通信（如任务审批通知）

#### 后端 (Server)
- **运行时**：Node.js
- **Web 框架**：Express 5
- **ORM**：Prisma (类型安全的数据库工具)
- **数据库**：SQLite (本地文件数据库，轻量便捷)
- **认证**：JWT (JSON Web Token) + bcryptjs (密码加密)
- **实时通信**：Socket.io

### 3. 核心数据模型 (Prisma Schema)
- **User**: 统一存储家长和孩子，通过 `role` 字段区分，支持层级关系 (`parentId`)。
- **Chore**: 定义家务任务及其对应的积分值。
- **Reward**: 定义可兑换的奖励及其消耗积分。
- **PendingAction**: 孩子提交的待审批家务记录。
- **InventoryItem**: 孩子已兑换但未使用的奖励库存。
- **History**: 记录积分变动的所有历史流水。

### 4. 关键业务流程
1.  **家长流程**：注册/登录 -> 添加孩子 -> 设置家务/奖励 -> 审批孩子提交的任务 (Approve/Reject)。
2.  **孩子流程**：登录 -> 查看任务 -> 完成任务 (提交审批) -> 获得积分 -> 兑换奖励。

### 5. 项目结构
- `client/`: 前端代码，包含页面 (`pages/`)、组件 (`components/`) 和 API 逻辑。
- `server/`: 后端代码，包含数据库模型 (`prisma/`)、迁移文件 (`migrations/`) 和 API 路由。
